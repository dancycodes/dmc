## What It Does
Renders the main layout for a cook's branded tenant website. When a visitor arrives at a cook's subdomain or custom domain, this feature provides the full page shell with the cook's selected theme applied (colors, fonts, border radiuses from F-011). The layout includes a responsive navigation bar, hero section placeholder, content sections (meals grid, about/bio, ratings summary, testimonials, schedule, delivery areas), and a footer with social links. Navigation items: Home, Meals, About, Contact. Auth links (Login/Register or user menu) are displayed in the header. All sections are scroll-anchored for single-page navigation.

## User Scenarios
### Scenario 1: First visit to a cook's site
A visitor navigates to `latifa.dm.test`. The tenant resolves via F-004, the cook's selected theme is applied (e.g., Modern theme with Poppins font and rounded corners), and the full landing page renders with the navigation bar, hero section, meals grid, about section, and footer.

### Scenario 2: Mobile browsing
A visitor opens the cook's site on their phone. The navigation collapses to a hamburger menu. All sections stack vertically. The layout is optimized for touch interaction and mobile viewports.

### Scenario 3: Authenticated client visit
A logged-in client visits the cook's site. The navigation shows their name/avatar with a dropdown (My Orders, My Favorites, Settings, Logout) instead of Login/Register links.

### Scenario 4: Theme customization applied
The cook has selected the "Neo Brutalism" theme with Inter font and sharp corners. The entire landing page reflects these choices: bold borders, no border radius, Inter typography throughout.

### Scenario 5: Section navigation
The visitor clicks "Meals" in the navigation. The page smoothly scrolls to the meals grid section. The URL hash updates to `#meals`. All section links work as anchor scrolls via Gale navigation.

## Business Rules
| Code | Rule |
|------|------|
| BR-126 | The tenant landing page renders ONLY on tenant domains (not the main domain) |
| BR-127 | The cook's selected theme (colors, fonts, border radiuses) must be dynamically applied to the entire page |
| BR-128 | Navigation must include: Home, Meals, About, Contact as scroll-anchor links |
| BR-129 | Auth state must be reflected: guest sees Login/Register; authenticated user sees their name with dropdown menu |
| BR-130 | The layout must be fully responsive: mobile-first design with hamburger navigation on small screens |
| BR-131 | All user-facing text must use `__('English text')` localization |
| BR-132 | The page must support light and dark mode, respecting the tenant's theme in both modes |
| BR-133 | Page sections render in order: hero, meals grid, about/bio, ratings summary, testimonials, schedule, delivery areas, footer |
| BR-134 | The page is publicly accessible without authentication |
| BR-135 | All interactions (navigation, section loading) use Gale; no full page reloads |

## Acceptance Criteria
- **Given** a visitor on a tenant domain, **when** the page loads, **then** the cook's branded landing page renders with all sections
- **Given** a cook with a selected theme, **when** the page loads, **then** the theme colors, fonts, and radiuses are applied
- **Given** a guest visitor, **when** viewing the navigation, **then** Login and Register links are displayed
- **Given** an authenticated user, **when** viewing the navigation, **then** their name and a dropdown menu are displayed
- **Given** a mobile viewport, **when** the page loads, **then** the navigation shows a hamburger menu and all sections stack vertically
- **Given** the visitor clicks a navigation anchor, **when** the click fires, **then** the page scrolls smoothly to the corresponding section

---

## Verification Steps
1. Visit a tenant domain and confirm the full landing page renders with all sections
2. Verify the cook's theme is applied (check font family, colors, border radius)
3. Test navigation anchor links scroll to correct sections
4. Test mobile hamburger menu opens and closes correctly
5. Verify guest sees Login/Register and authenticated user sees their menu
6. Toggle dark mode and verify all sections render correctly
7. Switch language and verify all text is translated

---

## Edge Cases & Exceptions
| Condition | Expected Behavior |
|-----------|-------------------|
| Cook has no theme configured | Default DancyMeals theme applied |
| Tenant is inactive | "Site unavailable" page shown (handled by F-004) |
| Cook profile is incomplete (no bio, no images) | Sections with no data show graceful empty states |
| Very long cook name in navigation | Name truncated with ellipsis |
| Visitor on main domain | Tenant landing page does NOT render; discovery page shown instead |

## UI/UX Notes
- Navigation bar fixed at top with semi-transparent background on scroll
